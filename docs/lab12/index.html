Lab 12 • Continuous Deployment (CD)

Тасралтгүй хүргэлт (CD) — Танилцуулга

CI/CD pipeline-аа өргөтгөж, туршилт амжилттай болвол автоматаар deploy хийх концепцийг харуулсан HTML тайлан.

CD гэж юу вэ?

Continuous Deployment (CD) нь CI-ийн дараах алхам бөгөөд build/test амжилттай болсны дараа автоматаар Staging/Production орчинд deploy хийх процессыг автоматжуулна.

  • Алдаа эрт илрүүлэхээс гадна гар deploy-ийн эрсдэлийг бууруулна.
  • Release cycle хурдтай, найдвартай болно.

CI/CD pipeline — 4 алхам

1 Код бичих

Git branch дээр хөгжүүлэлт, lint/test локалд.

2 CI

Push/PR үед build/test автоматаар. (GitHub Actions-д аль хэдийн бий).

3 CD

CI амжилттай бол auto-deploy (Heroku/Netlify/VPS).

4 Production

Monitoring/rollback бэлэн; Staging→Prod ялгаатай тохиргоо.

CD-ийн давуу тал

  • Release-ийг давтамжтай, бага эрсдэлтэй болгоно.
  • Хүний алдаа гарах боломжийг бууруулна.
  • Feature lead time багасна.

Deploy демо (загвар)

Энэ бол загварчилсан товч; бодит deploy-т GitHub Actions workflow ашигладаг.

CD-г хэрэгжүүлэх алхмууд

  1. CI дээр build/test-г найдвартай болгож, үргэлж ногоон болгох.
  2. Staging deploy job нэмэх (GitHub Actions → VPS/Heroku/Netlify).
  3. Secrets/ENV-ийг GitHub Secrets-д хадгалах.
  4. Prod deploy-г CI green нөхцөлтэй автоматаар эсвэл manual approve-тай болгох.
  5. Monitoring + rollback (versioned release) төлөвлөх.
